Ant tasks.

ScriptsExecuteTask

How to register

To use the task in a build file, the following taskdef declaration should be added: 

    <taskdef resource="antscriptella.properties" classpath="scriptella.jar"/>
Note: Additional JDBC drivers required for scripts execution should also be appended to classpath: 

    <taskdef resource="antscriptella.properties" classpath="scriptella.jar;hsqldb.jar"/>

Parameters

AttributeDescriptionRequired
fileThe script file to execute.Yes, unless a nested <fileset> element is used.
progressIf true shows execution progress.No, default value is false.
inheritAllIf true, pass all properties to the Script Executor. Defaults to false.No
The following attributes are not supported yet
forkif enabled triggers the class execution in another VM (disabled by default)No
maxmemoryMax amount of memory to allocate to the forked VM (ignored if fork is disabled) No

Nested Elements

The task supports <fileset> and <classpath> nested elements.

Examples

Executes script.xml file in the current directory: 
<execute/>
Executes sql.xml file in the current directory using Oracle classes12.zip on classpath: 
<execute file="sql.xml" classpath="classes12.zip"/>
Executes all xml files in db directory using hsqldb.jar and jconn2.jar on classpath: 
<execute>
    <classpath>
    <pathelement location="lib/hsqldb.jar"/>
    <pathelement location="lib/jconn2.jar"/>
    </classpath>
    <fileset dir="db" includes="*.xml"/>
    </execute>
